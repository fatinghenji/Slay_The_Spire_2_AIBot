# Phase 04 — QnA 模式本地知识检索与边界限制

## 已完成内容

### 1. 本地知识检索引擎已落地
已新增：

- `aibot/Scripts/Knowledge/KnowledgeSearchEngine.cs`

本阶段新增了一个轻量本地 `KnowledgeSearchEngine`，专门面向问答模式使用。当前实现特点：

- 不依赖外部检索服务
- 不依赖额外索引组件
- 直接复用现有 `GuideKnowledgeBase`
- 同时结合：
  - 卡牌条目
  - 遗物条目
  - 构筑条目
  - 角色攻略摘要
  - 核心机制摘要
  - markdown 摘要片段

当前搜索引擎会优先尝试识别问题中的：

- 卡牌名称
- 遗物名称
- 构筑名称
- 角色/玩法类问题
- 机制类问题

然后将匹配结果整理成一个本地答案返回给 `QnA` 模式。

---

### 2. `QnAModeHandler` 已真正接上本地问答链路
已更新：

- `aibot/Scripts/Agent/Handlers/QnAModeHandler.cs`

当前 `QnA` 模式已经不再只是返回占位文本，而是具备如下流程：

1. 接收玩家自然语言问题
2. 先判断是否属于游戏范围内问题
3. 对明显适合工具回答的问题优先调用现有 Tool
4. 对其余问题走 `KnowledgeSearchEngine`
5. 如果本地知识不足，再回退到当前对局分析摘要
6. 将过程写入 `AiBotDecisionFeed`

因此第四阶段完成后，`QnA` 模式已经具备了第一版真正可用的本地问答能力。

---

### 3. 问答模式已支持 Tool 优先回答
本阶段没有重复造轮子，而是复用了第二阶段已有的 Tool：

- `inspect_deck`
- `inspect_relics`
- `inspect_potions`
- `inspect_enemy`
- `inspect_map`
- `analyze_run`
- `lookup_card`
- `lookup_relic`
- `lookup_build`

这意味着像下面这类问题，现在会优先走现有能力：

- `看看我当前卡组`
- `当前敌人是什么情况`
- `这张卡牌 xxx 是什么`
- `遗物 xxx 有什么用`
- `当前有哪些推荐构筑`
- `现在局势怎么样`

这样做的好处是：

- 复用已有白名单能力
- 保持行为边界明确
- 避免问答模式和半自动模式出现两套割裂逻辑

---

### 4. 已加入游戏范围边界过滤
本阶段在 `QnAModeHandler` 中加入了轻量边界判断：

- 允许：杀戮尖塔2相关术语、卡牌、遗物、敌人、地图、机制、构筑等问题
- 拒绝：明显游戏外问题，如编程、新闻、天气、写邮件、简历、数学题等

当前拒答策略是：

- 若判断为游戏外问题，直接返回受限提示
- 不调用 Skill
- 不调用外部模型
- 不扩展到游戏外领域

这满足了当前阶段“让 Agent 保持游戏专属边界”的核心要求。

## 未完成内容

### 1. 仍未实现多轮会话记忆
当前 `QnA` 模式是单轮优先：

- 当前问题单独处理
- 不保留复杂上下文状态
- 不支持连续追问中的代词消解

例如：

- `这张牌怎么样？`
- `那它适合哪个构筑？`

第二句里的 `它` 目前还无法稳定追溯到上一轮实体。

### 2. 本地知识检索仍是轻量规则版
当前搜索引擎已经可用，但仍然是轻量本地实现，尚未加入：

- 更强的相关性排序
- 中英文别名词典
- 自定义知识库的结构化校验与分层检索
- 更细粒度 markdown section 索引

### 3. 问答答案还没有结构化引用来源展示
虽然内部已经保留了 `Sources` 概念，但当前 UI 还没有把来源列表直观显示在聊天窗口中。

### 4. 还未接入受限 LLM fallback
当前阶段严格遵循“本地优先、轻量优先”，因此没有在 `QnA` 模式中加入云端问答 fallback。

## 遇到的问题

### 问题 1：不能为了问答模式再复制一套查询逻辑
如果 `QnA` 模式完全独立实现卡牌/遗物/构筑查询，会导致：

- 与第二阶段 Tool 能力重复
- 白名单边界不一致
- 后续维护成本变高

### 解决方案
改为：

- Tool 型问题优先复用现有 `Registry`
- 只有 Tool 不适合回答时，再走知识检索引擎

这让问答模式自然建立在 Agent 既有能力之上。

---

### 问题 2：本地知识检索必须足够轻量
用户此前明确希望不要依赖额外内容，以保证 Mod 轻便易用。因此本阶段不能引入重量级向量检索或外部服务。

### 解决方案
本阶段采用：

- `GuideKnowledgeBase` 现有实体直接匹配
- 角色攻略与核心机制摘要复用
- markdown 摘要片段截取
- 当前对局分析作为 fallback

这样在不增加额外依赖的前提下，仍然能给出可用答案。

---

### 问题 3：边界过滤不能过于宽松
如果问答模式默认回答任何问题，就会违背“杀戮尖塔2专属 Agent”的定位。

### 解决方案
加入允许词与阻断词的本地判断：

- 优先拦截明显游戏外话题
- 只在检测到游戏相关术语时进入正常问答链路
- 对模糊问题保守处理

当前是第一版轻量边界，后续仍可继续强化。

---
